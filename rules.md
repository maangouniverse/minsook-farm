# 프로젝트 규칙 및 실수를 통한 재발 방지 가이드

본 문서는 개발 과정에서 발생한 실수와 그에 따른 재발 방지 조치 및 프로젝트 규칙을 기록하는 문서입니다.

## 1. 인지된 실수 및 문제 사례 (2026-06-18)
*   **상황**: 개발 중 로컬 검증 서버를 구동하기 위해 `npx -y http-server` 명령을 비동기 태스크로 실행하려 함.
*   **문제**: 사용자 환경에 Node.js 또는 `npx`가 설치되어 있지 않거나 환경 변수(PATH)에 등록되어 있지 않아 `CommandNotFoundException` 오류로 인해 백그라운드 명령이 실패함.
*   **원인**: 대상 OS(Windows)의 개발 환경 구성을 사전 확인하지 않고 특정 도구가 당연히 존재할 것이라 가정하고 실행한 설계 및 실행 실수.

## 2. 재발 방지 수칙
1.  **로컬 도구 가용성 사전 검증**:
    *   `npx`, `python`, `node`, `git` 등 외부 CLI 도구를 사용하는 명령을 실행하기 전에, 반드시 해당 도구가 실행 가능한지 가볍게 테스트 버전 조회 명령어(예: `npx -v`, `python --version`)를 먼저 실행하여 존재 여부를 확인한다.
2.  **대체 실행 방법 마련**:
    *   주요 도구가 존재하지 않을 경우를 대비해 대체 가능한 방안(예: Node.js가 없을 경우 Python 내장 서버 활용, 둘 다 없을 경우 로컬 파일 URL directly 사용 등)을 계획에 미리 반영한다.
3.  **환경 검사 우선원칙**:
    *   서버를 구동하거나 빌드 스크립트를 돌리기 전에 사용자의 OS 타입과 설치된 런타임 환경을 파악하여 이에 맞는 적합한 명령을 proposing 하도록 보수적으로 동작한다.
